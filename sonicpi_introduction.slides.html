Per què necessites pandas?¶

Pandas: anàlisis de dades en python¶

Per processament de dades en Python, la llibrera Pandas ha esdevingut essencial.

Qu s pandas?

  • Pots pensar en Pandas com arrays NumPy amb etiquetes per files i columnes, i un millor suport per a tipus de dades heterogenis... per es molt ms que aix
  • Tamb pots pensar en Pandas com els data.frame de R per per a Python.
  • Molt potent per treballar amb missing data, treballar amb sries temporals, per llegir i escriure les dades, per canviar la seva forma, agrupar, fusionar, etc.,

Documentaci: http://pandas.pydata.org/pandas-docs/stable/

Features principals¶

  • Suport fcil, rpid i flexible per a multitud de formats de dades
  • Mtodes per treballar amb missing data (.dropna(), pd.isnull())
  • Merging and joining (concat, join)
  • Grouping: groupby functionality
  • Reshaping (stack, pivot)
  • Manipulaci potent de sries temporals (resampling, timezones, ..)
  • Fcil visualitzaci en forma de grfics

Estructures de dades bàsiques¶

Pandas fa tot aix amb dos tipus d'objectes, construits sobre arrays de Numpy: l'objecte Series, i l'objecte DataFrame.

Series¶

Series bsicament es un contenidor per dades uni-dimensionals etiquetades. Es pot instanciar de la mateixa manera que els arrays NumPy:

In [8]:
s = pd.Series([0.1, 0.2, 0.3, 0.4])
s
Out[8]:
0    0.1
1    0.2
2    0.3
3    0.4
dtype: float64

Atributs de Series: index i values¶

Series incorpora el concepte de un index, el cual per defecte son nombres enters de 0 a N - 1

In [9]:
s.index
Out[9]:
RangeIndex(start=0, stop=4, step=1)

Pots accedir a la representaci internta del array numpy amb l'atribut .values:

In [10]:
s.values
Out[10]:
array([ 0.1,  0.2,  0.3,  0.4])

Podem accedir als valors de la serie fent servir l'index, com en els arrays de Numpy::

In [11]:

Out[11]:

DataFrames: Multi-dimensional Data¶

Un DataFrame es una estructura de dades tabular (objecte multi-dimensional que cont dades etiquetades) format per files i columnes. Similar a fulls de clculs, taules de bases de dades relacionals, etc.,.

Multiples objectes Series que comparteixen el mateix ndex

Una de les formes ms comunes de crear un dataframe es a partir de un diccionari de arrays o llistes.

Algunes notes sobre seleccionar dades¶

Una de les funcionalitats bsiques de pandas s l'etiquetat de files i columnes, per aix fa una mica ms complexe l'indexat en comparaci a numpy. Ara hem de diferenciar entre:

  • selecci per etiqueta
  • selecci per posici

Per un DataFrame, la indexaci bsica selecciona les columnes:

Seleccionar una columna:

In [43]:
provincies['superficie']
Out[43]:
provincia
Barcelona     7733
Girona        5905
Lleida       12172
Tarragona     6302
Name: superficie, dtype: int64

o mltiples columnes:

In [44]:
provincies[['superficie', 'densitat']]
Out[44]:
superficie densitat
provincia
Barcelona 7733 713.620846
Girona 5905 125.537680
Lleida 12172 35.097601
Tarragona 6302 125.386385

L'slicing accedeix a les files:

In [45]:
provincies['Girona':'Tarragona']
Out[45]:
habitants muntanya mes alta superficie densitat
provincia
Girona 0.741300 Puig Pedros 5905 125.537680
Lleida 0.427208 Pica d'estats 12172 35.097601
Tarragona 0.790185 Caro 6302 125.386385

Per un indexat ms avanat, tenim atributs extra:

  • loc: selecci per etiqueta
  • iloc: selecci per posici
